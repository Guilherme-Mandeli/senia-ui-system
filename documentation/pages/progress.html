Barras de Progreso (`progress`)

Normalización tipográfica y estética del elemento nativo HTML5 <progress> para reportar el avance de operaciones asíncronas, cargas y tareas.

Estados con Valores Determinados

Al definir los atributos value y max, la barra muestra de forma estática el porcentaje de avance correspondiente:

Valores Estáticos
<progress>[value="25"] <progress>[value="60"] <progress>[value="100"]
Porcentaje Bajo (25%)
Porcentaje Medio (60%)
Completado (100%)

Variantes de Color Semánticas

El framework ofrece modificadores BEM específicos para colorear la barra de progreso acorde al contexto cromático:

Clases de Color Semánticas
.c-progress--secondary .c-progress--success .c-progress--warning .c-progress--error
Secondary (Dorado Verde)
Success (Operación Exitosa)
Warning (Alerta de Capacidad)
Error (Fallo crítico o cancelación)

Estado Indeterminado (Carga Activa)

Al omitir el atributo value, la barra entra de forma automática en un estado indeterminado, proyectando una animación de barrido infinito de gran suavidad visual:

Barras de Carga Indeterminadas
<progress>:indeterminate .c-progress--success:indeterminate
Carga por Defecto (Primary)
Carga Exitosa (Success Indeterminate)

Código de Ejemplo

<!-- Barra determinada por defecto -->
<progress value="70" max="100"></progress>

<!-- Barra determinada con color secundario -->
<progress class="c-progress--secondary" value="45" max="100"></progress>

<!-- Barra indeterminada animada (cargando) -->
<progress></progress>

<!-- Barra indeterminada animada de error -->
<progress class="c-progress--error"></progress>
