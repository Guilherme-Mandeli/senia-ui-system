Arquitectura del Framework

Entiende los pilares de ingeniería CSS detrás de Senia UI: capas de cascada, modularidad atómica y compilación de alto rendimiento.

1. Capas de Cascada Ordenadas (@layer)

Uno de los mayores problemas históricos en grandes bases de código CSS es la "guerra de especificidad". Senia UI resuelve esto nativamente usando CSS Cascading Layers (capas de cascada).

Al declarar explícitamente el orden de las capas, garantizamos que las reglas de una capa posterior siempre prevalezcan sobre las anteriores, independientemente de la complejidad de sus selectores.

Orden de Prevalencia (de menos prioritario a más prioritario):

@layer reset, tokens, base, utilities, components, layouts, themes;
Capa (@layer) Ubicación del Código Responsabilidad
1. reset src/core/reset.css Resets modernos basados en Andy Bell y Josh Comeau. Normaliza el renderizado entre navegadores.
2. tokens src/tokens/*.css Definiciones cromáticas en HSL, escalas de espaciado, tipografía, bordes, sombras y z-index globales.
3. base src/base/*.css Reglas genéricas sobre selectores HTML directos (body, h1-h6, p, tables). Inicializa tipografías.
4. utilities src/utilities/*.css Clases atómicas de única responsabilidad (ej. .u-mb-vi o .u-text-center) para pequeños ajustes rápidos de maquetación.
5. components src/components/*.css Bloques web encapsulados construidos bajo la convención BEM Simplificada (ej. .c-button o .c-card).
6. layouts src/layouts/*.css Sistemas de cuadrícula globales, flexbox wrappers y contenedores estructurales de alto nivel (ej. .l-container).
7. themes src/themes/*.css Mapeo de variables de tema oscuro y variables contextuales para personalización de marcas.

2. Estructura de Directorios

El framework mantiene una separación limpia de responsabilidades organizando el código dentro de /src en módulos fáciles de mantener e importar: 

src/
├── core/
│   ├── layers.css      # Declaración inicial del orden de prioridad
│   └── reset.css       # Normalizaciones HTML básicas
├── tokens/
│   ├── colors.css      # Paleta HSL
│   ├── spacing.css     # Escalas de margin y padding
│   ├── z-index.css     # Apilamiento jerárquico
│   └── index.css       # Agregador de tokens
├── base/
│   ├── base.css        # Estilos HTML semánticos globales
│   └── index.css
├── components/
│   ├── button.css      # Componente Botón
│   ├── modal.css       # Modales estándar y nativos <dialog>
│   └── index.css
├── style.css           # Archivo de entrada agregador global

3. Herramientas y Pipeline de Compilación

Para garantizar que el CSS final esté optimizado para producción, utilizamos una arquitectura de tooling moderna:

  • Vite: Actúa como el servidor de desarrollo rápido y la orquesta del empaquetado.
  • PostCSS: Procesa el CSS nativo permitiendo el uso de imports anidados y estructuración modular.
  • Autoprefixer: Analiza el código CSS final y añade automáticamente prefijos de fabricante (vendor prefixes) basados en el soporte global actual de navegadores.
  • cssnano: Realiza optimizaciones agresivas eliminando comentarios innecesarios, espacios en blanco y minificando los valores para reducir al mínimo el peso final del archivo.
