Cascading Layers (@layer)

Controlamos la prioridad de cascada y evitamos la polución y la especificidad salvaje estructurando nuestros estilos en capas lógicas ordenadas de la siguiente manera: 

@layer reset, tokens, base, utilities, components, layouts, themes;

1. Reset

Andy Bell & Josh Comeau modern-normalize. Normalizaciones elementales y accesibilidad.

2. Tokens

Variables de diseño corporativas en formato HSL (Colores, Margen, Sombras, etc.).

3. Base

Configuración global de fuentes tipográficas, etiquetas nativas y resets tipográficos.

4. Utilities

Clases atómicas de baja especificidad (flex, display, text-align) para retoques rápidos.

5. Components

Módulos encapsulados estructurados bajo BEM Simplificado (.c-btn, .c-card, .c-input).

Convenciones del Sistema

  • Prefijos Claros: Separamos responsabilidades usando prefijos c- para componentes, l- para layouts, y u- para clases utility.
  • Baja Especificidad: No anidamos selectores innecesariamente (máximo 3 niveles). Evitamos el uso de IDs para aplicar estilos.
  • Variables HSL: La paleta cromática se define con componentes cromáticos H, S y L que permiten variaciones rápidas de hover e interactividad.

Empezar a Utilizar

Para compilar el framework para entornos de producción, ejecuta el comando de bundler para optimizar y minificar todo el CSS en un único archivo listo para distribución: 

pnpm run build

Los estilos generados se minificarán con cssnano y se auto-prefijarán con autoprefixer para garantizar una compatibilidad del 100% en todos los navegadores modernos del mercado.
